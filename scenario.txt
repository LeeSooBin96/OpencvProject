프로젝트 시나리오 
(서버에서 제품 검사) --이걸로 해야할듯
1. 클라이언트에서 업무 시작 -> 검사 제품 선택
2. 제품 스캔 -> 서버에 제품 번호(혹은 이름)와 함께 전송
3. 서버에서 제품 검사(제품 카운트) -> 결과 클라이언트에 전송
4. 2,3번 과정 반복
5. 업무 종료 시 서버에서 업무량 전송 -> 클라이언트 확인 후 프로그램 종료

시나리오 대로 다시 처음부터 서버 - 클라이언트 작성해보기

(클라이언트)
업무 시작 화면 - 구성 : 검사 제품 선택 콤보박스, 업무 시작 버튼
업무 시작 버튼 -> 서버에 연결 -> 제품명(혹은 제품 코드) 송신 -> 확인 메시지 수신
[제품 스캔 시작 -> 스캔본 서버에 전송 ->검사 결과 수신 -> 통과면 pass, 불량이면 NG ] 반복
업무 종료 버튼 -> 서버에 업무 종료 메시지 송신 -> 검사 카운트 수신 -> 출력 및 서버 연결 종료 ->종료


(서버)
프로그램 시작과 동시에 서버 오픈, 클라이언트 연결 대기
클라이언트 연결 수락 -> 제품명(혹은 제품 코드) 수신 -> 접속 사항 출력-> 확인 메시지 송신
[스캔본 수신 -> 제품 검사 -> 검사 결과 송신, 검사 카운트 저장 ] 반복
업무 종료 메시지 수신 -> 검사 카운트 송신 -> 클라이언트 연결 종료
필요 구성 - 접속한 클라이언트-제품명-검사카운트 출력하는 리스트

다중 클라이언트 염려해두고 코드 짜기 시작하자 
나중에 넣고 안돼서 코드 뒤엎는 것 보다 낫다.
